# P30页面 - 渠道分布（多图，单品牌×渠道占比）

## 页面概述

P30 页面展示“各品牌在不同渠道的占比”小图矩阵。每个小图对应一个品牌，图内的横轴为渠道类别（按统一顺序：Online News、Forum、Blog、X、Instagram、YouTube），纵轴是该品牌在该渠道的声量占该品牌全渠道声量的百分比（和约等于 100）。

## 图表配置

### 替换图表列表
- 本页包含大量小图（参见 `config.yaml` 的 `output.replace_charts` 列表），覆盖 P30 页面上所有品牌位的占位图。

### 配置参数
- **final_mode**: `updated` - 使用更新后的数据
- **axis_day_base**: `20300` - 日期轴基准值

## 数据源与字段映射

### 核心数据表
1. `neticle-v5-08.sqlite/mentions` 与 `keywords`、`sources`：
   - 使用 `mentions.createdAtUtcMs` 进行时间筛选（UTC 毫秒）。
   - 使用 `keywords.countryId` 与 `keywords.name` 做国家/品牌筛选与品牌归并（`lenovo%` 等）。
   - 使用 `sources.name` 做渠道归并，渠道映射来自全局 `config.yaml.channels`。

### 渠道映射
- 在全局 `config.yaml` 中维护：
  - `channels`: 原始 `sources.name` → 目标渠道（如 `forum`, `article`, `twitter` 等映射为 Forum/Online News/X 等）。
  - `fill_policy.channel_order`: 输出的统一渠道顺序。

## 核心算法实现

### 1. 单品牌×渠道占比 (`make_channel_distribution`)
功能：在给定时间和国家内，统计每个品牌在各个渠道上的 mentions 数；对每个品牌做行内归一化，输出百分比（保留 1 位小数）。

- 品牌归并：`keywords.name` 按全局 `filters.brands` 做 `lower(name) LIKE '<brand>%'` 归并。
- 渠道归并：`sources.name` 按 `config.yaml.channels` 归一到 6 个目标渠道。
- 时间过滤：`mentions.createdAtUtcMs` ∈ `[update.start_date, update.end_date)`。
- 国家过滤：`keywords.countryId = filters.countryId`。

### 2. 图表数据输出（类别图）
本页小图虽然是 `lineChart` 类型，但使用分类轴（非时间序列）。数据结构与柱/饼/面积一致：
```json
{
  "labels": ["Online News","Forum","Blog","X","Instagram","YouTube"],
  "series": [{"name": null, "values": [34.1, 22.7, 8.0, 12.3, 16.0, 6.9]}]
}
```

## 多图填充策略

### 小图排序与品牌分配
- 读取 `p30/config.yaml.output.replace_charts` 作为目标顺序。
- 依序将品牌循环分配给小图位（第 i 个小图使用 `filters.brands[i % len(brands)]`）。
- 每个小图写入该品牌的渠道分布百分比。
